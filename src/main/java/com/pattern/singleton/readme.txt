饿汉模式
    在示例之前，不管用户用都先new出来，避免线程安全问题
懒汉模式
    默认加载的时候不实例化，等需要的时候在实例化，延迟加载
注册登记式
    单例模式有饿汉模式、懒汉模式、静态内部类、枚举等方式实现，
    但由于以上模式的构造方法是私有的，不可继承，
    Spring为实现单例类可继承，使用的是单例注册表的方式（登记式单例）
    登记式单例实际上维护的是一组单例类的实例，将这些实例存储到一个Map(登记簿)中，
    对于已经登记过的单例，则从工厂直接返回，对于没有登记的，则先登记，而后返回


为了序列化与反序列化问题
    readResolve
    所有的单例模式都可以加上这个，这个是用来处理序列化与反序列化后对象不是同一个了的问题
    为了对象的重复利用
