代理模式：
    aop拦截器、拦截器、中介作用、解耦、增强
代理：
    静态代理、动态代理
    代理角色、被代理角色（目标对象），由被代理角色做决定
    静态代理与动态代理区别：
        静态代理：代理之前所以的东西都是已知的
        动态代理：所有东西都是未知的，运行的时候才加载
    动态代理：
        1、拿到被代理对象的引用，并且获取到他所有的接口，反射获取
        2、jdk Proxy类重新生成一个新的类，同事新的类要实现被代理类所有实现的接口
        3、动态生成java代码把新加的业务逻辑方法由一定的逻辑代码去调用（在代码中体现）
        4、编译新生成的java代码.class
        5、再重新加载到jvm中运行，
        这就叫字节码重组
     在jdk中$开头都是自动生成的
